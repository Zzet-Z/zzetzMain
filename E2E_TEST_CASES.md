## TC-01：首页浏览 + CTA 入口跳转

**目标：** 验证首页内容完整渲染，CTA 按钮触发 token 输入流程

**前置条件：** 无登录状态，打开全新标签页

### 步骤

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 访问 `https://zzetz.cn` | 页面加载，Hero 区"把你想要的网站，说出来。"标题可见 |
| 2 | 向下滚动查看完整首页 | "你不需要先会写 PRD"、"三步把模糊想法变成可执行方案"、"最后你会拿到什么"、底部 CTA 板块依次渲染 |
| 3 | 点击"开始梳理我的网站"CTA 按钮 | 弹出 token 输入对话框（或跳转 token 输入页） |
| 4 | 输入非法 token（如 `invalid-token-123`），提交 | 显示中文错误提示（如"无效的访问入口"），不跳转 |
| 5 | 输入合法 token，提交 | 跳转至 `/session/<token>`，session 页面开始加载 |

### 当前生产状态
⚠️ **BUG-01**：步骤 3 点击后按钮停在"正在准备梳理页..."永不结束，未弹出 token 输入框。
**修复方向：** `frontend/src/routes/home-page.tsx` CTA onClick 逻辑，移除匿名 `createSession()` 调用，替换为 token 输入 UI。

## TC-02：后台鉴权 + Token 签发

**目标：** 验证管理员鉴权隔离，token 签发后立即出现在列表

**前置条件：** 已知 `ADMIN_TOKEN`

### 步骤

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 访问 `https://zzetz.cn/admin` | "受控 Token 管理后台"标题与鉴权表单可见，TOKEN 列表显示"还没有可展示的 token" |
| 2 | 输入错误 admin token（`wrong-token`），点击"进入后台" | TOKEN 列表无数据，不泄露任何 token 信息（后端应返回 401） |
| 3 | 输入正确 admin token，点击"进入后台" | TOKEN 列表加载，显示总条数与 token 记录 |
| 4 | 在"访客备注"填写备注，点击"签发新 Token" | 列表顶部出现新 token，状态 `awaiting_user`，消息数为 1，备注匹配 |
| 5 | 点击新 token 的"查看"按钮 | 右侧详情面板展示：状态、消息数、附件数、创建时间、失效时间 |
| 6 | 点击"撤销 Token" | token 状态变为 `revoked`（或从列表移除），无法再用于访问 session |

### 验收标准
- 错误 admin token → 列表不返回数据
- 签发成功 → token 立即出现，消息数 = 1（欢迎语），状态 = `awaiting_user`
- 撤销后 → 访问对应 `/session/:token` 返回中文 404

## TC-03：访客凭 Token 进入聊天页

**目标：** 验证有效/无效 token 的不同渲染路径，以及首轮与修订轮的 UI 差异

**前置条件：** 已有合法 `awaiting_user` token（TC-02 签发），以及一个已 `completed` 的 token

### 步骤

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 访问 `/session/<invalid-token>` | 中文错误提示页面（如"访问入口无效"），不渲染聊天界面 |
| 2 | 访问 `/session/<new-awaiting-token>` | 3s 内加载完成，显示"CHAT-FIRST SESSION / 受控需求对话"，状态 `awaiting_user` |
| 3 | 验证首轮欢迎语 | LLM 欢迎语气泡在左侧，引导用户描述网站目标、目标用户、风格和参考案例 |
| 4 | 验证输入区 | 文本框、"选择文件"按钮、"发送"按钮均可交互 |
| 5 | 访问 `/session/<completed-token>` | 显示聊天历史，输入区禁用，顶部展示后续修订 Token |
| 6 | 访问 `/session/<revision-token>` | 顶部显示上一版摘要，欢迎语为"欢迎回来，这一轮我会基于上一版需求文档..." |

### 当前生产状态
⚠️ **BUG-02**：刚签发的新 token 访问 session 页面永久显示"正在加载..."，已有历史消息的 token 正常加载。
**修复方向：** 检查 `session-page.tsx` 轮询逻辑对初始 `draft`/`in_progress` 状态的处理。

## TC-04：发送消息 + Optimistic 回显 + 持久化

**目标：** 验证 F1 修复——消息发送后不被轮询覆盖，LLM 回复后完整持久化

**前置条件：** 已在有效 `awaiting_user` session 页面，LLM 欢迎语已渲染

### 步骤

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 在文本框输入"我想做一个摄影作品集网站，面向潜在商业客户" | 输入框内容正确显示 |
| 2 | 点击"发送"按钮 | 用户消息气泡**立即**出现在右侧（蓝色），发送按钮变为禁用 |
| 3 | 等待 3s（轮询间隔）| 用户消息气泡**仍然可见**，不因轮询拉取旧快照而消失（F1 验证点） |
| 4 | 验证 loading 态 | 左侧出现"思考中..."或 typing 占位气泡 |
| 5 | 等待 LLM 回复（生产约 30–100s） | 左侧出现 LLM 追问式中文回复，输入区重新可用 |
| 6 | 刷新页面 | 用户消息 + LLM 回复完整回放，顺序正确，内容不丢失 |
| 7 | 连续发送 2 条消息（第 1 条 LLM 回复前发第 2 条） | 两条用户消息均持续显示，不互相覆盖 |

### 验收标准
- 用户消息发送即显示，全程不消失
- LLM 处理期间有明确 loading 态
- 刷新后历史完整

## TC-05：图片上传 + 缩略图展示 + 错误反馈

**目标：** 验证 F4（缩略图）+ F2（错误具体原因）修复

**前置条件：** 已在有效 `awaiting_user` session 页面

### 成功路径

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 点击"选择文件"，选择一张 JPEG 图片（< 5MB） | 文件选择框打开 |
| 2 | 确认选择 | 附件区出现该图片的**缩略图**，而不只是文件名（F4 验证点） |
| 3 | 刷新页面 | 缩略图在历史回放中仍然可见 |
| 4 | 发送一条消息，等待 LLM 回复 | LLM 回复中提到"您上传了参考图"或引用附件信息（F3 验证点） |

### 错误路径（F2 验证点）

| # | 操作 | 预期结果 |
|---|------|---------|
| 5 | 上传 `.pdf` 文件 | 显示具体中文错误（如"不支持该文件格式，请上传 PNG / JPEG / WEBP"），不显示通用"上传失败" |
| 6 | 上传超过大小限制的图片 | 显示具体中文错误（如"文件超过大小限制"） |
| 7 | 在已 `completed` session 上上传图片 | 显示中文错误（如"该会话已结束，无法继续上传"） |

### 验收标准
- 合法图片 → 缩略图展示，刷新后回放
- 非法/超限/已结束 → 显示具体中文原因，不显示通用兜底文案

## TC-06：PRD 文档生成 + 会话完成 + 修订链

**目标：** 验证 F6（状态机正确推进）+ 完成弹框 + 后续修订 Token 全链路

**前置条件：** 已有充分对话的 `awaiting_user` session，LLM 已提议生成文档

### 步骤

| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 在对话中描述充分需求，等待 LLM 提议"是否生成最终需求文档" | LLM 给出明确的生成提议 |
| 2 | 回复"是的，请生成" | **Session 状态进入 `generating_document`**，不停留在 `awaiting_user`（F6 验证点） |
| 3 | 等待文档生成（约 30–100s） | 前端展示生成中提示，5s 轮询文档状态 |
| 4 | 文档生成完成 | 弹框出现："您的网站将于3-24小时内上线" |
| 5 | 验证 header 区 | 状态显示 `completed`，展示**后续修订 Token** |
| 6 | 点击弹框"确认" | 跳转回 `https://zzetz.cn` 首页 |
| 7 | 在后台查看该 token 详情 | 状态 = `completed`，文档状态 = `ready`，摘要非空，后续修订 TOKEN 字段有值，完成时间已记录 |
| 8 | 使用后续修订 Token 访问 `/session/<successor_token>` | 显示"欢迎回来"修订欢迎语 + 上一版需求摘要，输入区可用 |
| 9 | 在后台撤销后续修订 Token，再访问该 token 的 session | 中文错误提示，无法进入对话 |

### 验收标准
- 用户确认生成后 → 不停留在 `awaiting_user`（F6）
- 文档生成完成 → completed + 弹框 + 后续修订 Token
- 确认弹框 → 跳转首页
- 修订 Token → 修订会话，上一版摘要注入
